## 核心特性

### 1. 基于智能合约的发行

所有代币铸造和分发完全通过经审计的智能合约自动执行：

- 供应量仅在链上活动支持时增长
- 透明、可验证的代币经济模型
- 核心操作无需人工干预

### 2. 白名单系统

早期参与者可通过以下方式获得优惠价格：

- 积分资格认定
- 社区贡献奖励
- 战略合作伙伴配额
- 活动资格

### 3. 无许可参与

IRO 向所有参与者开放：

- 标准参与无需 KYC
- 可配置的单钱包认购限额
- 公开发售阶段平等参与权

### 4. 国库池

每个 RWA 代币维护一个由智能合约控制的专用国库：

**资金来源：**
- 初始募资收入
- 绑定曲线交易费用
- 底层资产的 APY 分配

**国库功能：**
- 支持地板价保证
- 资助投资者赎回
- 支持价格稳定机制

### 5. 地板价机制

每个 RWA 代币都有数学保证的最低价值：

```
地板价 = 国库总资产 ÷ 流通供应量
```

这确保了：
- 透明、可验证的最低代币价值
- 防止灾难性损失
- 投资者可预测的最坏情况

### 6. 下行保护

协议强制执行防止地板价下降的机制：

- **买入操作**：向国库添加资金，铸造新代币
- **卖出操作**：从国库返还资金，销毁代币
- **结果**：地板价只能上升，不能下降

## 通过 IRO 发行 RWA

### 步骤 1：连接钱包

连接 Web3 钱包（MetaMask、OKX 或兼容钱包）管理您的 RWA 发行。妥善保管私钥——钱包地址将成为链上资产发行方。

### 步骤 2：创建资产

使用以下参数定义您的 RWA：

| 参数 | 描述 |
|------|------|
| 资产名称 | 代币化资产的全称 |
| 代币符号 | 交易符号（如 "RWRE" 代表房地产） |
| 描述 | 底层资产的详细描述 |
| 网站 | 项目或资产网站 URL |
| 社交链接 | Twitter/X、Telegram、Discord（可选） |

### 步骤 3：配置发行参数

设置 IRO 的经济参数：

| 参数 | 描述 |
|------|------|
| 总供应量 | 最大代币供应量 |
| 团队分配 | 预留给团队/开发的百分比 |
| 解锁时间表 | 团队代币的释放时间线 |
| APY | 投资者年化收益率 |
| 奖励间隔 | 收益分配频率 |
| 软顶 | 发行所需的最低资金 |
| 硬顶 | 接受的最大资金 |
| 白名单期限 | 白名单阶段时长 |
| 公售期限 | 公开发售阶段时长 |

### 步骤 4：发行

确认参数后，IRO 合约部署完成，发行按配置的时间线开始。

## 投资者参与

### 白名单阶段

```typescript
// 检查白名单状态
const isWhitelisted = await gla.isWhitelisted(userAddress);

// 以优惠价格购买
const amount = ethers.parseUnits("1000", 6); // 1000 WeUSD
await weusd.approve(glaAddress, amount);
await gla.whitelistBuy(amount);
```

### 公开发售阶段

```typescript
// 向所有参与者开放
const amount = ethers.parseUnits("500", 6);
await weusd.approve(glaAddress, amount);
await gla.publicOfferingBuy(amount);
```

### 市场初始化后

```typescript
// 领取 RWA 代币
await gla.claim();

// 或在二级市场交易
const market = new ethers.Contract(marketAddress, MARKET_ABI, signer);
await market.buy(weusdAddress, amount, minOutput);
```

## 经济模型

### 国库支持的价值

每个 IRO 创建一个独立的微型经济体：

| 组件 | 功能 |
|------|------|
| 国库 | 持有支持资产，资助赎回 |
| 绑定曲线 | 算法确定买卖价格 |
| 税收机制 | 交易费用流入国库 |
| 地板价 | 由国库支持的最低保证价格 |

### 价格发现

RWA 代币价格由协议约束内的市场活动决定：

- **绑定曲线**提供算法定价
- **地板价**设定最低界限
- **市场供需**影响实际交易价格
- **国库增长**可提高地板价
